# T1000-3: Release readiness PR comment draft UI

## 目的
PLAN.v2 Phase 8 のうち、QMO Release Readiness Summary と PR/Issue/CI artifact link を紐付けた PR comment draft を GUI から生成できるようにする。

## スコープ
- 対象: Insights/QMO route の draft UI、web API client 呼び出し、CI artifact metadata import の UI 利用、関連テスト。
- 非対象: GitHub へのコメント投稿、GitHub API からの PR/Issue/Artifact 自動取得、artifact download/extract、draft 永続化。

## アプローチ
既存の `createReleaseReviewDraft` と `importCiArtifacts` を UI から呼び出す。QMO route は既に最新 run の QMO summary を表示しているため、同じ summary を入力にした card を追加する。PR/Issue/CI artifact は任意入力とし、CI artifact は T1000-2 の import API で既知 kind に正規化してから draft request に渡す。生成結果は Markdown として readonly 表示し、copy 操作だけを提供する。

## 検討した代替案
- GitHub API 連携で PR/Issue/artifact を自動取得する: 認証・権限・rate limit が絡み、T1000-3 の UI draft 範囲を超えるため不採用。
- qmo-summary.md を直接表示する: PR/Issue/CI artifact context を含められず、T1000-1/T1000-2 の schema/API を使わないため不採用。

## 影響範囲
- 変更ファイル: `apps/web/src/features/release-review-draft/ReleaseReviewDraftPanel.tsx`, `apps/web/src/routes/qmo.tsx`
- テスト: `apps/web/test/features/release-review-draft-panel.test.tsx`, 必要に応じて route test
- 既存機能への影響: QMO summary がない状態では draft UI は pending/empty 表示のみ。既存 QMO banner, Insights, Allure history には影響させない。
- マイグレーション要否: なし。

## テスト方針
- QMO summary がない/未生成/エラーの各状態で draft 生成 action が出ないことを確認する。
- PR/Issue 入力と CI artifact metadata から import API と draft API が正しい payload で呼ばれ、Markdown が表示されることを確認する。
- CI artifact が未入力の場合は import API を呼ばず、空 artifacts で draft を生成することを確認する。
- partial input は API 呼び出し前に UI error として止める。

## リスクと緩和策
- 外部 URL/Markdown 注入: UI は外部 URL を fetch せず、最終 validation は shared schema/API に委ねる。表示は React escaping と readonly textarea に限定する。
- 誤送信: GitHub への送信機能は実装しない。copy だけに限定する。
- 過剰実装: 複数 PR/Issue/artifact の管理 UI は作らず、現タスクでは 1 件ずつの入力に留める。

## 想定外の判断ポイント
QMO route は URL に runId を持たないため、既存の `useLatestQmoSummary` が返す最新 summary の `runId` を draft 対象にする。将来 `/qmo/:runId` が追加された場合は同じ panel props を run-scoped に差し替えられる。

## セルフレビュー記録
- `/pr-review-toolkit:review-pr` は本環境で利用不可のため、フォールバックとして要件/設計/コード/テスト/セキュリティ観点のセルフレビューを実施する。
- GitHub 送信・外部 URL fetch・artifact download は追加していない。
- PR/Issue/CI artifact の partial input は API 呼び出し前に止め、最終 validation は既存 shared schema/API に委ねる。
- draft Markdown は readonly textarea に表示し、copy 操作だけを提供する。
