# T800-2: Trace video screenshot artifact launch links

## 目的
PLAN.v2.md Phase 6 のうち、run に紐づく trace/screenshot/video を GUI から直接開ける導線を追加する。artifact viewer や Trace Viewer 再実装は行わず、Workbench は安全な launch link を提供する。

## スコープ
- 対象: run summary の failed test attachments、agent の evidence artifact 配信 endpoint、RunConsole の artifact link 表示。
- 非対象: trace/screenshot/video の収集設定 UI、Playwright Trace Viewer / UI Mode / Codegen 起動、HTML/Allure report viewer。

## アプローチ
`RunMetadata.summary.failedTests[].attachments` から `trace` / `screenshot` / `video` だけを link 化する。UI は attachment path を直接 href にせず、`/api/runs/:runId/evidence/:failureIndex/:attachmentIndex` に向ける。Agent は run metadata を再読し、指定 index の attachment が対象 kind か、実ファイルが project root 配下の regular file か、kind に合う拡張子かを検証してから返す。

## 検討した代替案
- UI で absolute path / file URL をそのまま表示: ブラウザで開けない環境があり、path leak と path traversal の境界が曖昧になるため不採用。
- trace.zip を GUI 内で解析表示: Trace Viewer の再実装に近づくため T800-3 へ分離。

## 影響範囲
- 変更ファイル: `apps/agent/src/routes/runs.ts`, `apps/web/src/features/run-console/RunConsole.tsx`
- テスト: `apps/agent/test/server.test.ts`, `apps/web/test/features/run-console.test.tsx`
- 既存機能への影響: summary に対象 attachment がない run では UI 表示なし。
- マイグレーション要否: なし。

## テスト方針
- Agent: project root 配下の screenshot を 200 で返し、root 外 path と kind/extension mismatch は 404 にする。
- Web: completed event と snapshot の summary attachment から trace/screenshot/video link が表示され、log/json/html attachment は表示されないことを確認する。

## リスクと緩和策
- Path traversal / symlink escape: `realpath` と project root containment で検証する。
- 機密 artifact 露出: 対象 kind と拡張子を trace/screenshot/video に限定し、raw path は href に含めない。
- 大きい動画/trace の読み込み: 今回は launch link のみ。ブラウザで開く明示操作時だけ agent がファイルを返す。

## 想定外の判断ポイント
Playwright JSON の attachment path は run dir 外の `test-results` 配下になり得るため、run dir ではなく project root containment を採用する。ただし root 外または symlink escape は拒否する。

## セルフレビュー記録
- `/pr-review-toolkit:review-pr` は本環境で利用不可のため、フォールバックとして要件/設計/コード/テスト/セキュリティ観点のセルフレビューを実施する。
- UI は attachment の raw path を表示・送信せず、runId と attachment index だけを API href に含める。
- Agent は kind allowlist、拡張子 allowlist、`realpath` による project root containment、regular file 確認を通過した artifact だけを配信する。
- `trace` / `screenshot` / `video` 以外の attachment は UI/API の両方で非公開とする。
