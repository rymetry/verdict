# T800-1: Run request controls for grep project retries workers and headed mode

## 目的
PLAN.v2.md Phase 6 のうち、GUI から Playwright run request に grep/project/headed/retries/workers を指定できるようにする。trace/screenshot/video、UI Mode、Trace Viewer、Codegen は後続タスクで扱う。

## スコープ
- 対象: `RunRequest` schema、Playwright CLI argument builder、default command policy、QA View の RunControls。
- 非対象: trace/screenshot/video artifact 制御、UI Mode/Trace Viewer/Codegen 起動、Playwright config の実評価。

## アプローチ
既存の `RunRequest` に `retries` と `workers` を追加し、非負整数/正整数の境界を schema で表現する。`projectNames` と `headed` は既存 schema/backend が受けられるため、UI から送信するだけに留める。CLI 生成では Playwright 公式の `--project <name>`、`--headed`、`--retries <n>`、`--workers <n>` を付与し、default command policy に pair flag と数値 validation を追加する。UI は現在の RunControls に入力欄/checkbox を足し、空欄は request へ含めない。

## 検討した代替案
- config summary から project option を自動候補化する: T900 系の Explorer UI と関心が混ざるため見送る。
- workers に percentage 形式を許可する: Phase 6 の初期 GUI としては数値指定で十分。Playwright 互換の拡張は必要になった時点で追加する。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/playwright/builder.ts`, `apps/agent/src/commands/policy.ts`, `apps/web/src/features/run-controls/RunControls.tsx`
- テスト: `apps/agent/test/builder.test.ts`, `apps/agent/test/policy.test.ts`, `apps/agent/test/server.test.ts`, `apps/web/test/features/run-controls.test.tsx`, `apps/web/test/hooks/use-start-run-mutation.test.tsx`
- 既存機能への影響: 既存 request は default/optional のまま互換。
- マイグレーション要否: なし。

## テスト方針
- shared/backend: retries/workers を含む command が builder と policy を通ること、負数/0/flag-like 値を拒否すること。
- agent route: `/runs` が不正な retries/workers を 400 にすること。
- web: RunControls が trim/split した project names、headed、retries、workers を request に含め、空欄は省略すること。

## リスクと緩和策
- CLI flag injection: schema と policy の両方で flag-like 値、pair flag の欠落、不正数値を拒否する。
- 過剰な UI 複雑化: Phase 6 初期範囲の controls だけ追加し、artifact/launch 系は後続へ残す。
- Playwright workers の仕様差分: まず正整数のみ許可し、percent 指定などは明示要件が出るまで入れない。

## 想定外の判断ポイント
`ProjectSummary` には Playwright project 一覧がまだ無いため、project names は comma-separated text input とする。候補表示は Config/POM explorer か inventory 補強の後続タスクで扱う。
