# T1500-8: Code generation workbench context injection

Status: implemented in this PR.

## Scope

T1500-8 wires the existing AI test generation gateway to the project-scoped
`.workbench/` knowledge base loaded by T1500-2:

- `.workbench/AGENTS.md` becomes generation context.
- `.workbench/rules/*.md` are passed as policy.
- `.workbench/skills/*/SKILL.md` are passed as reusable workflow guidance.
- `.workbench/hooks/*.{sh,toml,json}` are passed as policy context only.
- `.workbench/prompts/*.md` are passed as prompt references.

## Safety contract

Workbench documents are not passed to the AI raw. The agent first:

- keeps only project-relative paths,
- replaces the project root and user home directory with placeholders,
- applies the existing secret redactor,
- truncates long documents to bounded trailing text, and
- emits warnings when redaction or truncation occurs.

Hook content is never executed by this flow. It is included only so generation
can respect local policy before proposing a patch.
