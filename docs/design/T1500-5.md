# T1500-5: Test layer judgment

Status: implemented in this PR.

## Scope

T1500-5 introduces the Phase C layer judgment boundary:

- `LayerJudgment` records one recommendation per observed flow.
- `recommended` is one of unit, integration, contract, e2e, manual, or
  none-needed.
- Each judgment carries confidence, rationale, alternatives, risk, and evidence
  step ids.

## Design

The first implementation is a deterministic advisor over `AnnotatedScreenModel`.
It chooses:

- E2E for payment and authentication flows,
- integration for form-only flows,
- E2E for navigation flows,
- unit for isolated single-step unknown behavior, and
- manual when semantics are insufficient.

This gives T1500-6 a stable input shape without committing to an LLM prompt or
inventory duplicate-detection policy yet.

## Deferred

- LLM-backed judgment can replace the heuristic strategy behind the same schema.
- Existing inventory duplicate detection will decide `none-needed` after the
  matching semantics are defined.
- Public API wiring belongs to the first full pipeline integration task.
