# T600-3: Repair Review diff UI and approval states

## 目的
PLAN.v2.md Phase 4 のうち、AI proposed patch を UI 上で確認し、check / temporary apply / rerun / before-after comparison / approve-reject の判断状態を扱える Repair Review 体験を追加する。

## スコープ
- 対象: web API client、Repair Review UI component、AI Analysis への統合、状態遷移表示、diff 表示、API/コンポーネントテスト。
- 非対象: GitHub PR 作成、恒久 apply、Issue/PR コメント draft、generated test review。後続 T1000/T1100 系で扱う。

## アプローチ
既存の AI Analysis panel の proposed patch 表示を Repair Review panel に置き換える。patch text は preformatted diff として表示し、操作は明示ボタンで `check` → `apply temporary` → `rerun` → `load comparison` の順に進める。Approve は UI 上の review state として扱い、Reject は temporary patch が当たっている場合だけ reverse API を呼ぶ。状態は backend 永続化せず、T600-3 のスコープでは operator の現在セッション判断に限定する。

## 検討した代替案
- backend に approve/reject 永続状態を追加: 後続の PR/Issue/生成テスト連携と状態責務が絡むため今回は不採用。
- Developer View に統合: QA が AI analysis からそのまま判断できる流れを優先し、QA View 右列の AI Analysis 配下に置く。

## 影響範囲
- 変更ファイル: `apps/web/src/api/client.ts`, `apps/web/src/features/repair-review/RepairReviewPanel.tsx`, `apps/web/src/features/ai-analysis/AiAnalysisPanel.tsx`, 関連テスト。
- 既存機能への影響: proposed patch がない analysis では既存の empty message を維持する。Run / Failure Review / QMO route は変更しない。
- マイグレーション要否: なし。

## テスト方針
- 追加するテスト: patch check/apply/revert/rerun/comparison API client、Repair Review の happy path、dirty/check 失敗表示、reject 時 reverse 呼び出し、AI Analysis への proposed patch 統合。

## リスクと緩和策
- ユーザー変更破壊: UI は backend の dirty block に従い、apply 前に check を明示実行する。
- 誤 approve: Approve は恒久反映ではなく UI state のみとし、文言も "Mark approved" に留める。
- 操作順序混乱: ボタンの enabled 条件と状態 badge で現在の段階を示す。

## 想定外の判断ポイント
PLAN.v2 は approve/reject を示すが、恒久 apply や PR 連携はまだ backend contract がない。T600-3 では review state を UI 層に閉じ、T1000/T1100 で永続化・外部連携を扱える余地を残す。

## セルフレビュー結果
- `/pr-review-toolkit:review-pr` は本環境で利用不可のためフォールバックを採用。
- ユーザー視点: AI Analysis の proposed patch から直接 check/apply/rerun/compare に進めるが、恒久反映は行わない。
- 運用視点: comparison が未生成の間は 409 を pending として扱い、既存 run polling の作法と揃える。
- 攻撃者視点: patch 本文は client から API body に渡すだけで、UI 側は HTML として解釈せず pre/text として描画する。
