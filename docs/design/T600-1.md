# T600-1: Patch check and temporary apply API

## 目的
PLAN.v2.md §25 / Phase 4 のうち、AI proposed patch を `git apply --check` で検証し、dirty worktree を壊さずに一時適用・反転できる Agent API を追加する。

## スコープ
- 対象: patch request/response schema、Git 専用 command policy、patch file parser、dirty target check、`/patches/check` / `/patches/apply-temporary` / `/patches/revert-temporary` API、単体/API テスト。
- 非対象: before/after rerun comparison、Repair Review UI、approve/reject 状態管理、temporary worktree 方式。後続 T600-2/T600-3 で扱う。

## アプローチ
patch text は argv に載せず `CommandRunner.stdin` で `git apply` に渡す。`diff --git a/... b/...` から対象ファイルを抽出し、絶対パス・traversal・NUL を拒否する。適用前に `git status --porcelain -- <files>` を実行し、対象ファイルに未コミット変更があれば check/apply をブロックする。apply は必ず check 成功後にだけ実行し、revert は同じ patch を `git apply --reverse -` に渡す。

## 検討した代替案
- 自動 stash: PLAN.v2 で禁止されており、ユーザー変更の復元失敗リスクがあるため不採用。
- temporary worktree: 安全性は高いが T600-1 の範囲を超えるため、将来検討に留める。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/commands/policy.ts`, `apps/agent/src/git/patchManager.ts`, `apps/agent/src/routes/patches.ts`, `apps/agent/src/server.ts`, 関連テスト。
- 既存機能への影響: Playwright / Allure / AI policy は変更しない。Git policy は patch API 専用 runner からのみ使う。
- マイグレーション要否: なし。

## テスト方針
- 追加するテスト: patch parser の path validation、dirty target block、`git apply --check` 成功/失敗、temporary apply/revert、API の 200/409/400 応答。

## リスクと緩和策
- ユーザー変更破壊: target dirty block と reverse 専用 endpoint で緩和する。
- command injection: shell=false、argv allowlist、patch stdin 化、project-relative path validation で緩和する。
- 無関係ファイル revert: revert は同一 patch の reverse のみを実行し、他ファイルを触る操作は提供しない。

## 想定外の判断ポイント
PLAN.v2 の endpoint 一覧には revert も含まれる。T600-1 名は check/apply だが、一時適用 API として revert 不在は運用上危険なため同一 PR に含める。

## セルフレビュー結果
- `/pr-review-toolkit:review-pr` は本環境で利用不可のためフォールバックを採用。
- ユーザー視点: dirty target は 409/blocked diagnostics で自動 stash せず、revert API を明示的に提供する。
- 運用視点: patch text は audit log に載せず stdin 経由に限定し、Git 実行は専用 policy + 既存 audit handler に乗せる。
- 攻撃者視点: 実行可能ファイルを `git` に固定し、許可 argv shape と project-relative path validation で command/path injection を制限する。
