# T1100-3: Quality Gate enforcement for generated tests

## 目的
Phase 9 の完了条件である「AI 生成テストが diff review と Quality Gate を通る運用」を満たすため、生成テスト patch の承認を rerun 後の QMO/Quality Gate 結果で制御する。

## スコープ
- 対象: `RepairReviewPanel` の承認ポリシー拡張、AI test generation 経路での Quality Gate 必須化、rerun の QMO summary 取得、UI 表示、関連テスト。
- 非対象: Quality Gate 実行ロジックそのもの、profile 定義変更、CI 必須チェック変更、生成 patch の自動適用。

## アプローチ
既存 rerun は通常 run と同じ RunManager lifecycle を通るため、QMO summary / Quality Gate result も rerunId で取得できる。`RepairReviewPanel` に `approvalPolicy` を追加し、既存 AI analysis 経路は `comparison-only` のまま維持する。AI test generation 経路だけ `generated-test-quality-gate` を指定し、comparison 取得後に rerunId の QMO summary を取得する。summary outcome が `ready` の場合だけ Mark approved を有効化し、`conditional` / `not-ready` / summary pending は承認不可にする。

## 検討した代替案
- agent 側で generated-test 専用 approval endpoint を作る: 既存 UI state machine と重複し、T1100-3 の最小実装を超えるため不採用。
- すべての Repair Review に Quality Gate を必須化する: 既存 Phase 4 の AI analysis repair review 動線を変えるため不採用。

## 影響範囲
- 変更ファイル: apps/web/src/features/repair-review/RepairReviewPanel.tsx, apps/web/src/features/ai-test-generation/AiTestGenerationPanel.tsx, 関連テスト
- 既存機能への影響: 既存 `RepairReviewPanel` 呼び出しはデフォルト `comparison-only` のため承認条件は変わらない。
- マイグレーション要否: なし。

## テスト方針
- 既存 Repair Review は comparison 取得後に approve 可能なままであることを確認する。
- generated-test policy では QMO outcome が `ready` になるまで approve が disabled であることを確認する。
- generated-test policy で QMO outcome が `not-ready` の場合は承認できないことを確認する。
- AI test generation panel が generated-test Quality Gate policy を `RepairReviewPanel` に渡すことを統合表示で確認する。

## リスクと緩和策
- QMO summary 未生成時の誤承認: `fetchQmoSummary` が `null` を返す間は approve を disabled にする。
- 既存 repair review の挙動変更: policy を opt-in にし、既存呼び出しには適用しない。
- Quality Gate が未設定のプロジェクト: 既存 QMO の `ready` 判定に従う。Allure 未設定時は tests-only readiness として扱う既存仕様を尊重する。

## 想定外の判断ポイント
PLAN.v2 は Quality Gate enforcement の UI/API 境界を明示していない。既存 RunManager が rerun の QMO summary を生成する構造を活かし、承認 UI での enforcement に限定するのが最小で安全な実装と判断した。

## セルフレビュー結果
- 要件: AI test generation 経路の patch 承認は diff comparison 後、rerun の QMO outcome が `ready` になるまで無効化される。
- スコープ: 既存 AI analysis repair review は default policy のまま維持し、自動適用・自動承認は追加していない。
- セキュリティ: patch は既存の check/apply/rerun/revert API 境界を通り、生成物を無検証で実行・承認しない。
- 運用: QMO summary が未生成または `conditional` / `not-ready` の場合は明示表示し、承認ボタンを無効化する。
- 一時ファイル: 追加なし。
