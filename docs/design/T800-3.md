# T800-3: UI Mode Trace Viewer and Codegen safe launch commands

## 目的
PLAN.v2.md Phase 6 のうち、UI Mode、Trace Viewer、Codegen を Workbench 内で再実装せず、ユーザーが既存 Playwright CLI を安全に起動できるコマンド導線を追加する。

## スコープ
- 対象: Playwright launch command request/response schema、Agent の command builder と API、Run controls UI の command 表示・copy 操作、関連テスト。
- 非対象: GUI から外部ブラウザや Playwright プロセスを直接 spawn すること、Trace Viewer/Codegen/UI Mode の再実装、artifact download や外部 URL fetch。

## アプローチ
Agent は現在開いている project の package manager template から local Playwright binary 起動 prefix を再利用し、`playwright test --ui`、`playwright codegen [http(s) URL]`、`playwright show-trace <project-relative .zip>` の argv 配列だけを返す。生成後に専用 command policy で prefix、subcommand、operand を検証する。
Frontend は Run controls 内に補助 panel を追加し、生成された command を表示して clipboard にコピーできるようにする。これは明示的なユーザー操作を前提にした copy-only UX であり、Workbench は外部プロセスを起動しない。

## 検討した代替案
- Agent が `spawn` で UI Mode / Codegen / Trace Viewer を直接起動する: ローカル GUI 起動・外部ブラウザ起動の環境差と破壊的操作確認の境界が大きくなるため採用しない。
- trace.zip artifact を Workbench 内で解析表示する: Trace Viewer の再実装に近づき、PLAN.v2.md の Non-goal に反するため採用しない。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/commands/policy.ts`, `apps/agent/src/playwright/builder.ts`, `apps/agent/src/routes/projects.ts`, `apps/web/src/api/client.ts`, `apps/web/src/features/run-controls/RunControls.tsx`
- 既存機能への影響: Run request の実行経路は変更しない。追加 API は command template 生成のみ。
- マイグレーション要否: なし。

## テスト方針
- 追加するテスト: shared schema の launch request validation、Agent policy/builder/API の safe command validation、Web API client、Run controls UI の command 生成・copy-only validation。

## リスクと緩和策
- CLI option injection: schema と command policy の二重検証で leading flag、absolute path、path traversal、unsupported subcommand を拒否する。
- 外部起動の副作用: Workbench は command を表示・copy するだけで spawn しない。
- Trace path leakage: trace path は project-relative `.zip` のみ許可し、absolute path は API 入力で拒否する。

## 想定外の判断ポイント
PLAN.v2.md は「起動」と表現するが、外部プロセス起動は環境依存と確認対象になりやすい。PoC の安全性とローカルファースト方針を優先し、検証済み起動コマンドの提示を T800-3 の完了条件とする。
