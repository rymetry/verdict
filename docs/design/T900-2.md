# T900-2: POM and locator explorer read-only UI

## 目的
PLAN.v2 Phase 7 の「config、fixture、POM、locator の構造と保守リスクを可視化する」を満たすため、Developer View に read-only explorer を接続する。

## スコープ
- 対象: `config-summary` の POM heuristic 追加、Developer View の current project / config summary / inventory 接続、locator signal 表示、Web/Agent/schema tests。
- 非対象: POM source 本文表示、POM 編集、locator 修正、完全 AST 解析、Git diff/terminal/browser console の実データ接続。

## アプローチ
T900-1 の config summary scan に POM-like file の検出を追加し、project-relative path、class 名、locator call の短い sample だけを返す。UI は `/dev` で current project がある時だけ `fetchConfigSummary` と `fetchInventory` を読み、config/fixture/POM/spec を FileTree に、POM locator と test inventory `codeSignals` の locator/assertion を Locator panel に表示する。current project が無い時は ProjectPicker を表示し、fake placeholder data は使わない。
Source/Diff/Terminal/Console は今回の対象外なので空の read-only placeholder として残し、wire 済みの FileTree/Locator/Run metadata からは placeholder badge を外す。

## 検討した代替案
- `ts-morph` を導入する: T700-3 と同様に依存追加と実装範囲が大きく、現在の spike 粒度を超えるため採用しない。
- POM 専用 API を新設する: 既存 `config-summary` が config/fixture/auth の explorer source なので、Phase 7 の read-only 構造情報は同 API に集約する。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/project/configSummary.ts`, `apps/agent/test/configSummary.test.ts`, `apps/agent/test/sharedSchema.test.ts`, `apps/web/src/routes/dev.tsx`, `apps/web/src/features/developer-view/*`, `apps/web/test/features/developer-view.test.tsx`, `apps/web/test/routes/router.test.tsx`, `apps/web/test/api/client.test.ts`
- 既存機能への影響: `/dev` の static placeholder 表示を current project ベースの read-only explorer に置換する。QA/QMO/run pipeline は変更しない。
- マイグレーション要否: なし。

## テスト方針
- POM-like file と locator sample が project-relative に抽出されること。
- shared schema と Web client が `pomFiles` を検証すること。
- Developer View が project 未オープン時に ProjectPicker を出し、open 時に config/fixture/POM/spec/locator を実データで表示すること。
- Router direct access の `/dev` 挙動を更新すること。

## リスクと緩和策
- 誤検出/取りこぼし: `source: heuristic` として完全性を主張しない。
- 機密情報露出: source 全文は返さず、project-relative path、class name、locator line の短い expression に限定する。
- UI の silent fallback: production route から placeholder fixture import を削除し、loading/error/empty を route 側で分岐する。

## 想定外の判断ポイント
PLAN.v2 は Phase 7 で `ts-morph` 導入後の完全抽出を想定しているが、現行依存には未導入である。T900-2 は read-only UI 接続を優先し、POM/locator は bounded heuristic として扱う。

## セルフレビュー結果
- スコープ確認: POM / locator read-only explorer の接続に限定し、編集・自動修正・ソース本文表示は追加していない。
- セキュリティ確認: project-relative path、class name、短い locator/assertion expression のみ返し、absolute path と source 全文は返していない。
- 一時ファイル確認: `_scratch_` / `tmp_` / `debugger` は今回差分に含めていない。
- 検証: `pnpm --filter @pwqa/shared build`, `pnpm --filter @pwqa/agent test -- test/configSummary.test.ts test/sharedSchema.test.ts`, `pnpm --filter @pwqa/web test -- test/api/client.test.ts test/features/developer-view.test.tsx test/routes/router.test.tsx`, `pnpm typecheck`, `pnpm build`, `git diff --check`。
