# T1500-4: Screen model schema and comprehension layer

Status: implemented in this PR.

## Scope

T1500-4 extends the T1500-3 exploration draft into an annotated screen model:

- `ExplorationScreenModelDraft` remains the Phase A adapter output.
- `AnnotatedScreenModel` adds semantic annotations for steps and observed flows.
- The agent includes a first comprehension layer that maps explored DOM/action text
  to stable business semantics such as authentication, payment, form, navigation,
  and observable outcomes.

## Design

The first implementation is deterministic and local. It does not call an LLM.
That keeps the boundary testable while giving T1500-5/T1500-6 a structured input
instead of raw DOM snapshots.

The schema is intentionally additive. Existing T1500-3 adapters can continue to
emit draft screen models, and the comprehension layer can annotate them before
layer judgment or test plan generation.

## Deferred

- LLM-backed semantic annotation belongs behind the same schema after we have
  a stable prompt/evaluation loop.
- Existing test inventory duplicate detection is part of T1500-5 layer judgment.
- Persisting annotated artifacts from a public route belongs to the first full
  pipeline integration task.
