# T1500-3 Exploration Engine

## Status

Implemented foundation slice.

## Scope

T1500-3 introduces the agent-side exploration engine boundary for Phase 1.5-β.
It does not choose Stagehand or Browser Use as the permanent primary engine.
That product decision remains OQ-2 / OQ-D and needs an evaluation dataset.

## Runtime Contract

- `.workbench/workbench.json` may declare exploration provider order and command adapters.
- Default provider is `stagehand`; default fallback is `browser-use`.
- Provider IDs are open strings (`stagehand`, `browser-use`, or project-specific IDs), so the evaluation phase can add custom wrappers without a config migration.
- Providers run through `CommandRunner` only. The adapter command receives JSON on stdin and must return a screen model JSON object on stdout.
- Adapter executables are configured as bare command names, not filesystem paths; executable allowlists still belong to `CommandRunner`.
- The initial exploration command policy permits only project-relative Node/Python adapter scripts. It intentionally does not allow `node -e`, absolute scripts, package-manager indirection, or shell execution.
- The engine retries each configured provider up to `exploration.maxAttempts`, then falls back to the next provider.
- If no provider is available, the engine fails closed with `EXPLORATION_ADAPTER_UNAVAILABLE`.
- If every provider fails, the engine fails with `EXPLORATION_FAILED`.
- Screen model drafts use the minimal shared schemas in `packages/shared/src/exploration.ts`; T1500-4 owns semantic expansion and comprehension.
- Adapter output is sanitized before it is returned or persisted. Token-like values, sensitive query parameters, password input values, and local absolute paths are redacted.

## Artifact Contract

Successful exploration persists:

```text
.playwright-workbench/runs/<runId>/exploration.json
```

The public result exposes only the project-relative artifact path. Absolute
paths stay internal to the agent process.

## Deferred

- Comprehension-layer expansion of the shared Screen Model schema: T1500-4.
- UI/API route for invoking exploration: after the schema is stable.
- Real Stagehand and Browser Use SDK adapters: after the evaluation dataset
  fixes the primary/fallback decision.
