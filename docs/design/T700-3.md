# T700-3: Locator assertion and Allure metadata extraction spike

## 目的
PLAN.v2 Phase 5 の「locator/assertion、Allure metadata を QA 向けに表示する」ため、Test Inventory に spec source 由来の静的 signal を追加する。

## スコープ
- 対象: inventory の source-text heuristic enrichment、shared schema の code signal、agent tests。
- 非対象: `ts-morph` 導入、完全な TypeScript AST 解析、Developer View への接続、POM 解析、実行時 DOM locator 状態。

## アプローチ
Playwright `--list` JSON を引き続き test inventory の主ソースにし、取得済み spec path を project root 内に限定して read-only に読む。source file は行単位で走査し、`test.step` / `allure.step` を操作ステップ、`expect(...)` を期待結果、`getByRole` などの locator 呼び出しと `allure.*` metadata 呼び出しを `codeSignals` として記録する。各 signal は line number と短い expression のみを返し、source 全文や外部 path は返さない。
完全 AST ではなく spike として bounded heuristic に留め、後続で `ts-morph` に置き換え可能な schema を先に固める。

## 検討した代替案
- `ts-morph` を追加して AST 解析する: runtime dependency 追加と実装範囲が大きく、T700-3 の spike 粒度を超えるため採用しない。
- UI まで同時に接続する: T700-2 で QA metadata 表示は完了済み。locator/allure signal の Developer View 接続は別タスクに分離する。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/project/inventory.ts`, `apps/agent/test/inventory.test.ts`, `apps/agent/test/sharedSchema.test.ts`, `apps/web/src/features/test-inventory/TestInventoryPanel.tsx`, `apps/web/test/features/test-inventory-panel.test.tsx`
- 既存機能への影響: inventory API response に optional `codeSignals` と enriched `qaMetadata.steps/expectations` が追加される。Playwright 実行・run pipeline は変更しない。source が `static-analysis` に変わっても title 由来の redundant purpose は UI で重複表示しない。
- マイグレーション要否: なし。

## テスト方針
- `test.step` と `allure.step` が `qaMetadata.steps` へ反映されること。
- `expect(...)` が `qaMetadata.expectations` と assertion signal へ反映されること。
- locator 呼び出しと Allure metadata 呼び出しが `codeSignals` に project-local line number 付きで入ること。
- shared schema が optional `codeSignals` を検証すること。
- `static-analysis` 補強後も test title 由来の重複 purpose が smoke test の strict locator を壊さないこと。

## リスクと緩和策
- 誤検出/取りこぼし: `source: static-analysis/allure-metadata` として heuristic であることを明示し、完全性を主張しない。
- 機密情報露出: project root 外の file は読まず、source 全文ではなく該当呼び出し行の短い expression のみ返す。
- 大きな spec file の負荷: source file size cap を設け、超過時は warning にする。

## 想定外の判断ポイント
PLAN.v2 は `ts-morph` を将来候補としているが、依存が未導入であり、T700-3 は「spike」として最小の read-only enrichment に限定する。完全 AST 化は signal schema が安定した後に行う。
