# T1100-2: Generated test diff review integration

## 目的
Phase 9 のうち、AI test generation gateway の `proposedPatch` を既存の diff review / temporary apply / rerun / comparison 導線へ接続する。

## スコープ
- 対象: QA View の AI test generation UI、web API client 呼び出し、生成結果の plan/evidence/risk/filesTouched 表示、既存 `RepairReviewPanel` への patch 受け渡し、関連テスト。
- 非対象: Quality Gate enforcement、生成 patch の自動適用、Playwright MCP/Test Agents の直接起動。これらは T1100-3 または後続明示タスクの責務。

## アプローチ
T1100-1 の `POST /runs/:runId/ai-test-generation` を web から明示実行する `AiTestGenerationPanel` を追加する。operator は mode、objective、任意 targetFiles を入力し、結果は `AiTestGenerationResponseSchema` で検証済みの値として表示する。`proposedPatch` がある場合だけ既存 `RepairReviewPanel` に渡し、patch check/apply/rerun/compare の既存安全経路を再利用する。

## 検討した代替案
- `AiAnalysisPanel` に生成フォームを混ぜる: failure classification と test generation は操作目的が違い、状態も別になるため不採用。
- 生成 patch を受信後に自動 check する: operator の明示操作を維持する Phase 4 の設計に反するため不採用。

## 影響範囲
- 変更ファイル: apps/web/src/features/ai-test-generation/AiTestGenerationPanel.tsx, apps/web/src/routes/qa.tsx, apps/web/test/features/ai-test-generation-panel.test.tsx
- 既存機能への影響: 既存 AI analysis / Repair Review の API と state machine は変更しない。
- マイグレーション要否: なし。

## テスト方針
- runId がないときは生成実行ボタンを出さないことを確認する。
- mode/objective/targetFiles を送信し、生成 result を表示することを確認する。
- proposedPatch がある場合に `RepairReviewPanel` の diff review が表示されることを確認する。
- API 失敗時に alert と console error が出ることを確認する。

## リスクと緩和策
- 生成 diff の過信: UI は diff を表示し、既存 Repair Review の明示 check/apply/rerun/compare 経路に限定する。
- 不正 targetFiles: web 側は空白/カンマ/改行を配列化するだけに留め、最終検証は shared schema と agent API に任せる。
- UI の状態混線: analysis と generation は別コンポーネント・別 mutation にして、runId 変更時に結果をリセットする。

## 想定外の判断ポイント
PLAN.v2 は “diff review に流す” とだけ定義しているため、既存 Repair Review パネルを再利用するのが最小実装と判断した。Quality Gate 合否を承認条件にする処理は T1100-3 へ残す。
