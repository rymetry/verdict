# T500-2: AI CLI adapter policy and analysis API

## 目的
PLAN.v2.md Phase 3 / §26 の AI Analysis のうち、T500-1 で作った redacted context を AI CLI に渡し、validated な `AiAnalysisOutput` として API から返す。

## スコープ
- 対象: Agent 側の AI CLI 実行 adapter、Claude Code 用の専用 command policy、`POST /runs/:runId/ai-analysis` API、結果 JSON の run directory への保存、単体/API テスト。
- 非対象: Web の AI Analysis View、AI patch の適用、Codex/Gemini/custom adapter、source snippet / locator 抽出の追加。これらは後続タスクで扱う。

## アプローチ
T204/T205 の Allure policy と同じく、AI CLI は既定の Playwright command policy から分離する。初期 adapter はローカル help で非対話・JSON Schema 出力を確認できた Claude Code のみに限定し、stdin で prompt + redacted context を渡す。CLI stdout は shared schema で検証し、CLI wrapper JSON の `result` に JSON 文字列が入る形と、直接 schema object が出る形の両方を許容する。API は既存の `failure-review` endpoint と同じ run loading 経路を使い、生成結果を `.playwright-workbench/runs/<runId>/ai-analysis.json` に保存する。

## 検討した代替案
- Codex/Gemini も同時対応: Codex は `exec` の stdin と JSONL は確認できたが output schema の永続ファイル指定が必要で、Gemini はローカル CLI が無いため、今回の policy に含めない。
- prompt を argv に入れる: audit log に context が残るため不採用。`CommandRunner` に stdin を追加し、argv/audit から context を外す。

## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/commands/policy.ts`, `apps/agent/src/commands/runner.ts`, `apps/agent/src/ai/cliAdapter.ts`, `apps/agent/src/routes/runs.ts`, `apps/agent/src/server.ts`, 関連テスト。
- 既存機能への影響: Playwright / Allure runner の既存 argv policy は変更しない。`CommandSpec.stdin` は任意項目なので既存 caller はそのまま動作する。
- マイグレーション要否: なし。

## テスト方針
- 追加するテスト: AI policy が Claude Code の許可 argv のみ通すこと、runner stdin が child process に渡ること、AI adapter が stdout JSON を schema 検証すること、API が context 生成・adapter 実行・結果保存を行うこと。

## リスクと緩和策
- AI CLI への secret 流出: T500-1 の redaction/context builder を唯一の入力源にし、argv/audit には context を載せない。
- AI CLI のファイル操作: Claude Code は `--tools ""` と `--bare` を使い、adapter policy もその argv のみに固定する。
- CLI 出力揺れ: shared `AiAnalysisOutputSchema` で検証し、不一致は 502 として fail closed する。

## 想定外の判断ポイント
ローカル `claude --help` で `--print`, `--output-format json`, `--json-schema`, `--tools ""`, `--bare` を確認した。`codex exec --help` は確認したが、今回必要な single JSON object output は追加検証が必要なため後続対象に回す。
