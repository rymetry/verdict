# T900-1: Config and fixture explorer read-only API

## 目的
PLAN.v2 Phase 7 の「config、fixture、POM、locator、storageState、重複/脆弱locator候補を可視化する」の入口として、Playwright config と fixture 構造を UI が読める read-only API で提供する。

## スコープ
- 対象: shared schema、agent の `GET /projects/:projectId/config-summary`、config/fixture の read-only heuristic summary、web API client、関連テスト。
- 非対象: UI 表示、POM/locator 詳細解析、storageState/auth risk の評価、config の実行・編集。

## アプローチ
既存 scanner と同じく config を実行せず、ファイルテキストを上限付きで読む heuristic に限定する。config から reporter 名と `use.trace/screenshot/video` の静的値を抽出し、fixture は project root 配下を上限付きで走査して `fixtures` ディレクトリ、`*.fixture.*`、`test.extend` を含む小さなソースファイルを列挙する。node_modules / .git / run artifact ディレクトリは走査対象外にし、API は編集・実行を一切しない。

## 検討した代替案
- Playwright config を import/evaluate する: ユーザーコード実行と副作用のリスクがあるため不採用。
- ts-morph で fixture graph を構築する: T900-1 の read-only API 粒度を超えるため不採用。後続の explorer / risk タスクで検討する。
- storageState の中身を読む: token 値を扱うリスクがあり、T900-3 の専用タスクに分離する。
## 影響範囲
- 変更ファイル: `packages/shared/src/index.ts`, `apps/agent/src/project/configSummary.ts`, `apps/agent/src/routes/projects.ts`, `apps/web/src/api/client.ts`, 関連テスト
- 既存機能への影響: 既存 project open / inventory / run API は変更しない。
- マイグレーション要否: なし。
## テスト方針
- config summary が reporter と use option を heuristic で抽出することを unit test する。
- fixture scan が fixture file と `test.extend` signal を project-relative path で返すことを unit test する。
- route が未 open project を 404 にし、open project の config summary を返すことを server test で確認する。
- web client が schema parse 経由で config summary を返すことを確認する。
## リスクと緩和策
- ユーザーコード実行: config は import せずテキストのみ読む。
- 大規模 project 走査コスト: directory / file count と file size に上限を設け、超過は warning にする。
- 機密情報露出: file content や storageState 内容は返さず、relative path と検出 signal だけ返す。
- 動的 config の取りこぼし: heuristic source と warning で明示し、誤検出より false negative を優先する。
## 想定外の判断ポイント
PLAN.v2 は config summary の具体 field を定義していない。既存 API Design に `GET /projects/:projectId/config-summary` があるため、その endpoint 名を採用し、UI が必要な最小情報だけを構造化する。
## セルフレビュー結果
- 要件: `GET /projects/:projectId/config-summary` が config reporter/use option と fixture-like file signals を read-only で返す。
- スコープ: UI、POM/locator、storageState/auth risk は後続タスクへ分離し、T900-1 は API contract と heuristic summary に限定した。
- セキュリティ: config は import/evaluate せず、file content や token 値は返さない。project-relative path と signal のみ返す。
- 運用: config / source file / directory / fixture count に上限を持たせ、超過や unreadable directory は warning にする。
- 一時ファイル: 追加なし。
