# T1100-1: AI test planning and generation gateway schema

## 目的
Phase 9 の土台として、Playwright MCP/Test Agents 相当の planner/generator/healer 結果を Workbench の diff review に渡せる structured schema と agent API を追加する。

## スコープ
- 対象: shared schema、既存 AI CLI adapter の test generation gateway、run-scoped API、web API client、関連テスト。
- 非対象: 生成 diff の UI 統合、patch check/apply/rerun 連携、Quality Gate enforcement。これらは T1100-2/T1100-3 の責務。

## アプローチ
既存の `buildAiAnalysisContext` を再利用し、run/failure/log context は既存 redaction 境界を通す。新しい request は `planner` / `generator` / `healer` mode、objective、project-relative targetFiles を受け取り、Claude CLI には stdin と JSON Schema で渡す。AI はファイル編集・コマンド実行・tool 利用を禁止し、結果は `plan`、`proposedPatch`、`filesTouched`、`evidence`、`risk`、confidence、human decision flag に限定する。

## 検討した代替案
- Playwright MCP や外部 Test Agent を直接起動する: tool 権限・認証・外部仕様依存が大きく、T1100-1 の gateway schema を超えるため不採用。
- 既存 AI analysis schema に詰め込む: failure classification と test generation は目的が異なり、後続 Quality Gate 判断も異なるため別 schema にする。

## 影響範囲
- 変更ファイル: packages/shared/src/index.ts, apps/agent/src/ai/cliAdapter.ts, apps/agent/src/routes/runs.ts, apps/web/src/api/client.ts, 関連テスト
- 既存機能への影響: 既存 AI analysis endpoint の request/response は変更しない。
- マイグレーション要否: なし。

## テスト方針
- shared schema が mode/objective/targetFiles と output shape を検証することを確認する。
- CLI adapter が generation prompt を stdin で渡し、JSON Schema output を検証することを確認する。
- API が completed run から redacted context を生成し、gateway response を返すことを確認する。
- web client が endpoint response を schema 検証することを確認する。

## リスクと緩和策
- AI の過剰権限: `--tools ""` と prompt で編集・コマンド実行を禁止し、出力 schema のみを受け取る。
- path/secret leakage: 既存 redacted analysis context を再利用し、targetFiles は project-relative かつ traversal/flag injection/Unix absolute/Windows absolute を拒否する。
- 生成 patch の過信: T1100-1 では適用しない。T1100-2 の diff review と T1100-3 の Quality Gate を必須経路にする。

## 想定外の判断ポイント
PLAN.v2 の Phase 9 は MCP/Test Agents を名指しするが、現行コードには該当 adapter がない。既存 Claude Code adapter を gateway として拡張し、外部 agent の直接統合は後続の明示タスクに残す。
