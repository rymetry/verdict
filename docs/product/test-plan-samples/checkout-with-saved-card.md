## Intent (PM)
保存済みクレジットカードを持つユーザは、カード情報を再入力せずに 3 クリック以内で checkout を完了できるべき。

### なぜこれが重要か
- ファネルの最重要瞬間における friction を低減する
- conversion 目標: saved-card ユーザの checkout 完遂率 65%
- 直近 incident: 2026-03 に Stripe API upgrade で saved-card flow が壊れ、24h 売上に影響

### Out of scope
- 新規カード入力 (`checkout-with-new-card` で別途)
- 3DS 認証 (`checkout-with-3ds-card` で別途)
- サブスクリプション商品 (`subscription-checkout` で別途)

## Steps (AI 探索結果)

### 前提
- Test user fixture: `users.with_saved_card` (Stripe customer ID `cus_test_xxx`、保存カード 1 枚)
- Test items: `items.standard_10usd` ($10.00 USD 商品)

### Flow
1. **Login** — `users.with_saved_card` で sign in (`auth-flow` skill 利用)
2. **Navigate** — `/cart` に遷移 (URL 確認: `await expect(page).toHaveURL("/cart")`)
3. **Add item** — `items.standard_10usd` を cart に追加 (`data-testid="add-to-cart-button"`)
4. **Verify cart total** — `data-testid="cart-total"` が `$10.00` 表示
5. **Click pay button** — `data-testid="checkout-pay-with-saved-card"`
6. **Verify modal amount** — `data-testid="checkout-confirm-amount"` が `$10.00` (High Risk: ここで amount 不整合が起きうる)
7. **Confirm** — `data-testid="checkout-confirm-button"` クリック
8. **Wait for Stripe webhook** — `await page.waitForURL(/\/orders\/[a-z0-9-]+$/)` (`payment-flow` skill 利用)
9. **Verify order** — order page で:
    - `data-testid="order-status"` === `"paid"`
    - `data-testid="order-amount"` === `"$10.00"`
    - `data-testid="order-items"` 配下に `items.standard_10usd` が 1 件

### click 数
3 click (step 5, 7 が user click。step 3 は preparation で外せる)
